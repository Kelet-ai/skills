# Kelet Integration Transcript — docs-ai (FastAPI + pydantic-ai)

**App context:** FastAPI backend using pydantic-ai. Sessions stored in Redis (session_id generated server-side, returned in X-Session-ID response header). No React frontend — plain HTML. Deployed on Fly.io.

---

## Welcome Banner

Kelet integration for docs-ai. Here's what we're setting up:

- **Trace** — every pydantic-ai LLM call and tool invocation captured automatically
- **Session** — groups all turns of a conversation for RCA correlation
- **Signal** — tips at meaningful moments (thumbs, edits, retries)
- **Synthetic** — platform evaluators running on every session automatically (no code)
- **Project** — docs-ai-assistant: one project for this docs Q&A use case

- `uv.lock` present — package manager: uv
- No React, no `@kelet-ai/feedback-ui`

### 2. Entrypoint
- `app/main.py` — `lifespan()` context manager → `kelet.configure()` goes here (or module level)
- `src/settings/__init__.py` had redundant `kelet_api_key` / `kelet_project` fields — removed (SDK reads env vars directly)

### 3. LLM Call Sites
- `src/routers/chat.py` → `_run_agent_stream()` — SSE streaming with `chat_agent.iter()`
- `GET /chat` — stateless, no session (correct — skip `agentic_session` here)
- `POST /chat` — session-based streaming (primary integration point)
- One flow, one project

### 4. Session Tracking
- `session_id` = UUID4 generated by `create_session()` in `src/cache/__init__.py`
- Stored in Redis with 30-min TTL
- Returned as `X-Session-ID` response header
- **`agentic_session()` REQUIRED** — app owns the session ID; pydantic-ai doesn't know it
- Session semantics: correct — UUID4 changes per conversation, resets on expiry
- **STREAMING NOTE:** `agentic_session()` must wrap entire generator body including `[DONE]` sentinel — trailing spans silently lost otherwise

### 5. Existing Feedback UI
- None — plain HTML frontend
- No thumbs/rating/edit hooks
- **VoteFeedback: N/A** (no React)

### 6. Deployment
- Kubernetes (Helm, EKS via ArgoCD)
- `k8s/charts/docs-ai/templates/deployment.yaml` — `KELET_API_KEY` from K8s Secret `docs-ai-kelet`, `KELET_PROJECT` from ConfigMap ✓
- `.env` for local dev — needs `KELET_PROJECT` added
- **No fly.toml found** (app description said Fly.io but actual deployment is Kubernetes — going with what's in the repo)

### Architecture Diagram
```
Browser / curl
      │  POST /chat {message, session_id?}
      ▼
FastAPI (app/main.py)
  kelet.configure() ← startup
      │
      ▼
POST /chat endpoint (src/routers/chat.py)
  ├─► get_session(redis, session_id) or create_session()
  │         session.session_id = UUID4 (Redis, 30-min TTL)
  └─► StreamingResponse(_run_agent_stream(...))
              │
              ▼
    async with kelet.agentic_session(session_id=session.session_id)
              │
              ▼
    chat_agent.iter(message, deps=deps)
      ├─► tool: search_docs(query) → BM25 over cached docs
      └─► tool: get_page(slug) → full doc page
              │
              ▼
    SSE stream → client
    X-Session-ID: <session.session_id> header
```
