**What Kelet is:** An AI detective that investigates AI app failures — not a dashboard. It reasons over traces and signals to find root causes and suggest fixes.

**The detective model:** Kelet sees two things:
- **Traces** — automatic recording of every LLM call, tool use, latency, tokens. No code needed.
- **Signals** — tips you drop at meaningful moments. *"Signals are tips — 'something went wrong here.' Traces are the scene. Kelet follows the evidence."*

A thumbs-down means *start looking here*, not *this session failed*. Retries say *the user wasn't satisfied*. More deliberate tips = narrower investigation = faster fix.

🔍  PHASE 0b · AGENTIC WORKFLOW + UX MAPPING
══════════════════════════════════════════════════

> 🧠 **What we're doing:** Mapping failure modes before proposing signals. Kelet clusters spans by failure pattern — every failure mode found here becomes a signal candidate in 0c. Signals without a failure map are guesses.

### Workflow

**Steps:**
1. User submits question via POST /chat (or GET /chat for stateless)
2. Session resolved from Redis (or created new)
3. `chat_agent.iter()` invoked with message + history
4. Agent calls `search_docs(query)` → BM25 search over indexed docs
5. Agent optionally calls `get_page(slug)` → full page retrieval
6. Agent synthesizes answer from retrieved content
7. SSE stream delivered to client; history persisted to Redis

**Where it can go wrong:**
- **Wrong retrieval** — BM25 returns irrelevant chunks; agent synthesizes from wrong context
- **Hallucination** — agent invents API names or features not in retrieved docs
- **Deflection** — agent says "I don't know" when the answer exists in the docs
- **Multi-turn context loss** — session expires mid-conversation; user loses history
- **Tool over-calling** — agent calls `search_docs` + `get_page` redundantly, inflating latency
- **Off-topic drift** — agent answers general questions outside doc scope

### UX (User Perspective)

- Plain HTML frontend — no React, no vote UI, no edit hooks in the UI
- User sees streamed text response
- User can only signal dissatisfaction implicitly: abandon (close tab), rephrase (send another message), or go elsewhere
- No explicit feedback mechanism exists in the current UI
